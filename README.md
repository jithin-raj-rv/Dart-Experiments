
# Dart Experiments — Flutter Learning Projects

**Overview**
- **Purpose**: A collection of small Flutter projects used to learn core Flutter concepts, state management, UI patterns, packages, and platform tooling.

**Projects**
- **basic_counter**: Simple counter app demonstrating widget structure and basic state. See [basic_counter/README.md](basic_counter/README.md).
- **basic_navigation**: Examples of navigation and routing patterns. See [basic_navigation/README.md](basic_navigation/README.md).
- **hello_world**: Minimal starter app for Flutter fundamentals. See [hello_world/README.md](hello_world/README.md).
- **hive_learn**: Experiments using Hive local storage. See [hive_learn/README.md](hive_learn/README.md).
- **Learning BLoc and Lottie**: Projects exploring BLoC patterns and Lottie animations (subfolders: `learnin_bloc`, `mustache_counter`).
- **Learning Getx**: Examples using the GetX package for state and routing (folder: `getx_learning`).
- **login_ui**: UI-focused login screens and authentication flows. See [login_ui/README.md](login_ui/README.md).
- **print_value**: Small exercises and example apps (to_do_app, etc.). See [print_value/README.md](print_value/README.md).
- **rock_paper_scissor**: Game app sample demonstrating input, game logic, and simple animations. See [rock_paper_scissor/README.md](rock_paper_scissor/README.md).
- **testing notification**: Local notification experiments (folder: `local_notificaiton`).
- **weather_app**: Demo app that fetches weather data and shows practical API usage. See [weather_app/README.md](weather_app/README.md).

**Getting Started**
- **Prerequisites**: Install Flutter SDK and platform tooling for Android/iOS. Confirm `flutter doctor` is clean.
- **Open a project**: `cd <project-folder>` then `flutter pub get`.
- **Run on device/emulator**: `flutter run -d <device-id>`.
- **Run tests**: `flutter test` from the project root.

**Per-Project Notes**
- **Dependencies**: Each project lists dependencies in its `pubspec.yaml` (run `flutter pub get`).
- **Platform files**: Android, iOS, web, macOS, Windows and Linux folders may be present; pick the platform you target.

**Contributing / Usage**
- **Explore**: Open any project folder in VS Code or Android Studio to inspect code and run it.
- **Reuse**: Copy smaller examples into new projects to experiment safely.
